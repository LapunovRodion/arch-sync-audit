# Архитектура проекта

`arch-sync-audit` проектируется как учебная CLI-утилита на Rust. Архитектура должна оставаться простой: сначала чистые структуры данных и тестируемая логика, затем подключение реальных системных команд.

## Основная идея

Программа работает с двумя состояниями системы:

- ожидаемое состояние из файла `system.toml`;
- текущее состояние машины, на которой запущена команда.

Затем программа сравнивает эти состояния и показывает различия.

## Предполагаемые модули

```text
src/
  main.rs
  cli.rs
  profile.rs
  collect.rs
  compare.rs
  plan.rs
  report.rs
```

## Ответственность модулей

`main.rs`:

- точка входа программы;
- вызывает разбор CLI;
- передает управление нужной команде;
- связывает чтение файла, сбор текущего профиля, сравнение и вывод.

`cli.rs`:

- описывает аргументы командной строки;
- содержит команды `export`, `check`, `plan`;
- использует `clap`.

Отдельного `cli.rs` пока нет: CLI-структуры находятся в `main.rs`, чтобы проект оставался проще на учебном этапе.

`profile.rs`:

- содержит структуру `SystemProfile`;
- содержит типы данных, которые можно сериализовать через `serde`.

`collect.rs`:

- собирает текущее состояние системы;
- читает `$SHELL`;
- запускает `pacman`;
- проверяет известные config paths;
- запускает `systemctl` для system/user services;
- при ошибке `systemctl --user` печатает warning и возвращает пустой список user services.

`compare.rs`:

- сравнивает ожидаемый профиль и текущий профиль;
- не читает файлы;
- не запускает системные команды;
- не печатает результат;
- возвращает структуру различий.

`plan.rs`:

- превращает различия в безопасный текстовый план действий;
- не выполняет команды;
- не устанавливает пакеты;
- не меняет систему.

`report.rs`:

- превращает `ProfileDiff` в человекочитаемый отчет для `check`;
- не читает файлы;
- не запускает системные команды;
- не меняет систему.

## Архитектурный принцип

Главное правило: отделять чистую логику от внешнего мира.

Чистая логика:

- сравнение профилей;
- генерация списка недостающих пакетов;
- генерация текста плана;
- генерация текстового отчета.

Внешний мир:

- чтение файлов;
- запись файлов;
- переменные окружения;
- запуск `pacman`;
- запуск `systemctl`.

Такой подход нужен, чтобы важную логику можно было тестировать без реальной Arch Linux системы.

## Текущий поток данных

`export`:

1. `main.rs` вызывает функции из `collect.rs`.
2. Собранные данные складываются в `SystemProfile`.
3. Профиль сериализуется в TOML через `toml::to_string_pretty`.
4. TOML записывается в файл.

`check`:

1. `main.rs` читает профиль из TOML.
2. `collect.rs` собирает текущий профиль.
3. `compare.rs` строит `ProfileDiff`.
4. `report.rs` превращает diff в текстовый отчет.

`plan`:

1. `main.rs` читает профиль из TOML.
2. `collect.rs` собирает текущий профиль.
3. `compare.rs` строит `ProfileDiff`.
4. `plan.rs` превращает diff в безопасный план ручных действий.
